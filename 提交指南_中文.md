## 🎬 Demo Video 怎么做?

**时长:** ≤5分钟 (3-4分钟更好)

**内容:**
1. **介绍 (30秒)**
   - Context8是什么?
   - 解决什么问题? (agents没有记忆)

2. **架构 (1分钟)**
   - 展示架构图 (可以截图README里的)
   - 解释: MCP协议 → 混合搜索 → SQLite存储

3. **实际演示 (2分钟)** ⭐ 最重要
   - 打开Cursor或VS Code
   - 展示Context8已安装 (MCP servers列表)
   - **保存一个错误解决方案** (让AI调用save-error-solution)
   - **搜索类似错误** (让AI调用search-solutions)
   - 展示搜索结果(相似度分数)

4. **技术细节 (1分钟)**
   - 展示数据库结构 (可以打开 `~/.context8/solutions.db`)
   - 解释hybrid search (semantic + sparse)
   - 展示性能指标 (sub-second search)

5. **成果 (30秒)**
   - 15+客户端集成
   - 10K solutions测试通过
   - npm package链接

**录制工具:**
- macOS: QuickTime Screen Recording
- Windows: OBS Studio (免费)
- 在线: Loom

**上传到:**
- YouTube (unlisted) 推荐 ✅
- Google Drive (设置为"任何人可查看")
- Vimeo

## ❓ 常见问题

### Q1: 我一定要做Demo Video吗?
**A:** 虽然标注为可选,但**强烈建议做**。因为:
- 容易得分 (5分)
- 展示你的项目很有说服力
- 3-5分钟很快就能录完

### Q2: AgentBeats链接怎么办?
**A:** Context8是工具不是agent,所以:
1. 先与导师确认是否需要
2. 如果不需要,在Q11写:
   ```
   Not applicable - Context8 is an MCP server tool for agent memory,
   not a white/green agent itself.

   Alternative metrics:
   - npm package: https://www.npmjs.com/package/context8-mcp
   - 15+ MCP client integrations verified
   - Performance benchmarks in research report
   ```

### Q3: Research Report一定要2页吗?
**A:** 1-2页都可以。模板比较长(~2500字),你可以:
- 保持完整版 (2页,2-column layout)
- 删减到1页 (精简版)
建议至少1.5页,内容充实更好

### Q4: 我英语不好,报告要全英文吗?
**A:** 是的,学术报告建议用英文。但是:
- 我已经写好了完整的模板
- 你只需要根据实际情况微调
- 技术术语已经都写好了
